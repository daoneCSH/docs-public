# Operations Overview

DADP 운영은 단순 헬스체크 모음이 아니라, 제어면 상태, 실행면 상태, 연동 경계 상태를 분리해서 보는 운영 모델이다.

## 주요 운영 대상

| 컴포넌트 | 운영 의미 | 대표 확인 항목 |
|----------|-----------|----------------|
| Hub | 정책 원본과 운영자 진입점 | UI/API 가용성, 정책/키 상태, 연동 등록 |
| Engine | 공통 런타임 실행면 | health, round trip, sync-check |
| Aggregator | 선택적 관측 경계 | 수집과 조회 가용성 |
| Wrapper | 애플리케이션 연동 경계 | 인스턴스 등록, 매핑, 로컬 반영 |
| DB UDF | 데이터베이스 연동 경계 | 설치 오브젝트, ACL, Engine 호출 |

## 기본 점검 순서

1. 프로세스 또는 컨테이너 상태 확인
2. 최근 로그 확인
3. Hub 접근과 제어면 상태 확인
4. Engine round trip 확인
5. 영향을 받은 연동 경계 확인

## 운영 규칙

- 제어면 장애와 실행면 장애를 분리한다.
- 로그 보존 없이 재기동부터 하지 않는다.
- 정책 변경과 인프라 변경의 영향을 따로 판단한다.
- 특정 연동 경계 문제를 전체 플랫폼 문제로 일반화하지 않는다.
