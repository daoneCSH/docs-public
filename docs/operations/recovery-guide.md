# Recovery Guide

장애 복구의 목적은 빨리 재기동하는 것이 아니라, 영향 범위를 줄이면서 원인을 잃지 않고 정상 상태를 회복하는 것이다. DADP 복구는 제어면, 실행면, 연동 경계를 같은 순서로 보지 않는 것이 중요하다.

## 공통 복구 순서

1. 장애 범위 확인
2. 최근 변경 사항 확인
3. 로그 보존
4. 제어면과 실행면 상태 분리 확인
5. 필요한 순서로 복구 수행
6. 기능 검증 수행

## 기본 복구 체크

- Hub 로그인 또는 제어면 접근 가능 여부
- 정책과 키 조회 가능 여부
- Engine health
- 단건 암복호화 round trip
- 필요한 경우 batch 또는 특정 연동 경계 검증

## 권장 복구 해석

### 제어면 중심 장애

정책 조회, UI 진입, 인스턴스 등록이 먼저 깨졌다면 Hub와 동기화 경로를 우선 본다.

### 실행면 중심 장애

여러 연동 경로에서 동시에 실행 실패가 발생하면 Engine과 공통 실행면을 먼저 본다.

### 연동 경계 중심 장애

특정 애플리케이션이나 특정 DB만 실패하면 Wrapper 또는 DB UDF 경계를 먼저 본다.

## 복구 완료 기준

- 주요 서비스가 정상 기동한다.
- 제어면 접근과 실행면 round trip이 모두 정상이다.
- 치명 오류 로그가 계속 발생하지 않는다.
- 문제를 재현하던 경로에서 동일 증상이 사라진다.
