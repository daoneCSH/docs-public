# Hub Operations

Hub 운영의 목적은 정책 원본, 키 메타데이터, 연동 매핑, 운영자 접근 경계를 안정적으로 유지하는 것이다.

## 일상 점검 항목

- Hub 로그 상태
- 최근 정책과 키 변경 이력
- 연동 인스턴스 등록 상태
- 동기화 상태
- 운영 UI와 제어면 API 접근성

## 권장 점검 순서

1. Hub 로그 확인
2. 정책과 키 상태 확인
3. 연동 인스턴스와 매핑 상태 확인
4. Engine 동기화 상태 확인
5. 필요 시 런타임 round trip 확인

## 장애 해석

Hub 문제는 보통 다음과 연결된다.

- 정책이 안 보이거나 갱신되지 않음
- 연동 인스턴스 등록이 실패함
- 운영자 UI 또는 API 접근이 불안정함
- 동기화 상태가 오래된 값으로 유지됨

반대로 암복호화 실행 실패가 먼저 보이면 Hub만의 문제인지 Engine까지 영향이 간 문제인지 구분해야 한다.
