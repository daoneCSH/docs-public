# Hub

Hub는 DADP 제어면의 중심 모듈이다. 정책과 키 메타데이터를 정의하고, 연동 인스턴스를 등록하고, 운영자 UI와 제어면 API를 제공한다.

## 주요 책임

- 정책과 마스킹 정책 관리
- 키 메타데이터 관리
- 연동 인스턴스 등록과 매핑 관리
- Engine 동기화 제어
- 운영 UI와 API 제공

## 보유하는 것과 보유하지 않는 것

### Hub가 보유하는 것

- 정책 원본
- 키 메타데이터 원본
- 연동 인스턴스 식별 정보
- 운영 상태와 관리 설정

### Hub가 직접 하지 않는 것

- 암복호화 실행
- 데이터면 런타임 처리
- 데이터베이스 내부 SQL 실행

## 운영 해석

Hub 장애는 보통 다음 범주에 먼저 영향을 준다.

- 새 정책 배포
- 연동 등록과 변경
- 운영자 가시성
- 동기화 상태 판정

따라서 Hub 문제가 발생했다고 해서 즉시 런타임 실행 전체 실패로 해석하면 안 된다. 반대로 정책 변경이 반영되지 않는다면 먼저 Hub 원본 상태와 동기화 경로를 봐야 한다.

## 운영자가 주의할 점

- Hub URL과 운영자 진입 URL을 혼동하지 않는다.
- 정책 원본 변경과 런타임 반영 시점은 구분해서 본다.
- 연동 인스턴스 관리와 실행 오류 진단은 같은 화면에서 이루어질 수 있어도 같은 문제 범주는 아니다.
