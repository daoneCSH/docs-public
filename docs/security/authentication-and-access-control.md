# Authentication and Access Control

DADP는 사용자 인증과 권한 부여를 분리해서 다룬다. 인증은 누가 접근하는지 확인하는 절차이고, 권한 부여는 그 사용자가 무엇을 할 수 있는지 결정하는 절차다.

## 사용자 인증

- 연결형 환경에서는 연결형 인증 경로를 통해 사용자 신원이 성립한다.
- 에어갭 환경에서는 Hub가 로컬 인증 경계를 제공한다.
- 공개 문서는 인증 동작과 운영 의미를 설명하며, 내부 인증 토폴로지의 상세 구현에는 의존하지 않는다.

## 권한 부여

Hub 기능 접근은 Hub 역할 모델을 기준으로 평가된다.

대표 예시는 다음과 같다.

- 정책 조회와 수정 권한
- 키 메타데이터 조회와 회전 권한
- 사용자와 역할 관리 권한
- 운영 설정과 모니터링 접근 권한

인증 성공은 권한 허용과 동일하지 않다.

## 보호되는 경로

| 경로 유형 | 설명 |
|-----------|------|
| 공개 경로 | 로그인, 헬스, 초기 진입, 복구 보조 경로 |
| 운영자 보호 경로 | 인증과 권한 검사를 모두 거치는 제어면 경로 |
| 서비스 연동 경로 | 운영자 UI와 다른 신뢰 가정을 사용하는 서비스 간 경로 |

## 응답 해석

| 상황 | 일반적 의미 |
|------|-------------|
| `401 Unauthorized` | 인증이 성립하지 않음 |
| `403 Forbidden` | 인증은 되었지만 권한이 부족함 |

## 운영상 해석

- 로그인 문제와 역할 문제를 같은 장애로 취급하지 않는다.
- 제어면 UI 접근 문제와 서비스 간 호출 문제를 같은 인증 문제로 묶지 않는다.
- 연결형과 에어갭 환경은 같은 로그인 화면을 제공하더라도 운영 해석이 다를 수 있다.
